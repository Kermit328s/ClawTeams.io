# 板块三：工作流层

> 开发优先级：**P2 — 依赖板块一（基础设施）和板块二（团队大脑意图图谱）就绪后启动**
> 关联章节：v2.1 第四章 4.3（工作流生成器）、场景二（营销视频全链路）

---

## 1. 板块职责

工作流生成器是核心自研组件——一个**编译器**，把意图图谱（声明式"想要什么"）编译成 Temporal（命令式"怎么做"）。用户永远不需要学习 Temporal，不需要懂工作流编排，不需要写任何代码。

---

## 2. 四个子模块

### ① 图谱解析器（Graph Parser）

**开发顺序：第一步。先支持简单顺序和并行，不支持条件。**

识别节点类型：
- 目标节点（Goal）
- 任务节点（Task）
- 决策节点（Decision）
- 人工节点（Human）

识别边类型：
- 顺序依赖（A完成后才能做B）
- 并行关系（A和B同时进行）
- 条件依赖（满足条件X才触发B）
- 聚合关系（A和B都完成才触发C）
- 回环关系（B不合格回到A重做）— 来自场景二质检回环

### ② 执行策略引擎（Execution Planner）

**开发顺序：第三步。加入能力匹配和路径优化后，龙虾协作才真正成立。**

- 能力匹配：每个任务需要什么能力，哪个龙虾能承接
- 执行路径优化：哪些任务可以并行，关键路径是什么
- 风险识别：哪些节点是瓶颈，哪些失败会阻塞全局

### ③ Temporal 代码生成器（Workflow Compiler）

**开发顺序：第二步。把解析结果跑起来，验证基本执行链路。**

翻译规则：
```
图谱节点       → Temporal Activity
顺序依赖       → await activity
并行关系       → Promise.all / asyncio.gather
条件依赖       → if/else分支
聚合关系       → 等待所有完成
人工节点       → Temporal Signal（工作流暂停等待）
回环关系       → 条件循环
```

### ④ 变化监听器（Change Listener）

**开发顺序：第四步。最后做，最复杂。前三步跑通后再处理动态调整。**

```
微小变化（优先级调整） → 动态更新Activity参数，不中断执行
中等变化（新增依赖）   → 在下一个检查点注入
重大变化（核心意图）   → 暂停工作流，重新生成，人工确认后生效
```

## 3. AI 驱动链路

### 完整链路

```
用户自然语言输入
→ AI解析意图，输出结构化JSON（不是自然语言）
→ AI生成执行计划（能力匹配 + 路径优化）
→ AI生成Temporal代码
→ 验证层（静态分析 + Schema验证 + 沙箱试跑）
→ Temporal执行引擎
```

### 验证顺序（逐步叠加）

```
第一步：只用AI做自然语言→图谱（验证语义理解准确率）
第二步：加入AI执行计划生成（验证能力匹配准确率）
第三步：加入AI代码生成（此时已有足够验证数据）
```

每一步验证 AI 可靠性后再推进，三层叠加的错误极难调试。

## 4. 意图变化的缓冲机制

```
意图图谱（高频变化）
    ↓
意图暂存区（缓冲）
├── 变化合并：10分钟内多次变化合并为一次
├── 变化分级：微小/中等/重大
└── 检查点机制：在任务自然边界处消化变化
    ↓
工作流（稳定执行）
```

检查点设置在：任务完成时、进入等待状态时、人工审批前、里程碑节点。

若意图变化速度远超检查点频率，系统主动提示：**意图尚未稳定，建议先稳定意图再继续执行。**

## 5. 工作流节点间的档案交接

来自场景二的关键发现：档案库不只是存储，是工作流节点之间的**数据总线**。

```
节点A完成
  → 成果存入资产档案库（带标记：类型、批次、关联任务）
  → 存入动作触发 artifact_uploaded 事件
  → 下游节点B通过档案库链接访问内容
  → B的执行结果同样写回档案库
```

每个工作流节点的输入和输出都通过档案库进行，形成完整的可追溯链：

```
A.output → 档案库 → B.input → B.output → 档案库 → C.input
```

## 6. 工作流模式示例（场景二：营销视频链路）

```
①人工拍摄（Human节点）
  约束从意图图谱向下传递：拍什么、什么风格、什么计划
  产出上传至档案库 → 触发事件
        ↓
②龙虾质检（AI节点）
  通过档案库链接访问视频内容
  判断结果：
    不合格 → 回环到①（LOOP_BACK边）
    合格 + 改进建议 → 认知写入 + 前进
    合格 → 前进
        ↓
③视频剪辑（混合节点）
  从档案库拉取素材，剪辑成品写回档案库
        ↓
④多平台分发（多AI节点并行，PARALLEL_WITH边）
  各平台龙虾并行：裁剪画幅、配文案、发布
        ↓
⑤数据追踪（定时AI节点）
  按时间周期采集：点击率、浏览率、评论
  参与评论互动
  数据写回档案库
        ↓
⑥反馈闭环（认知触发）
  数据回传到上游 + 意图图谱
  指导下一轮工作侧重
  底层假设得到验证或推翻
```

## 7. 与其他板块的接口

| 对接板块 | 接口内容 |
|---------|---------|
| 团队大脑 | 读取意图图谱、接收意图变更事件、写回工作流状态 |
| 连接层 | 通过事件总线下发任务给龙虾、接收龙虾状态上报 |
| 前端 | 工作流状态实时推送（Temporal → 事件 → WebSocket → 地图） |
| 基础设施 | 依赖 Temporal 基础环境、档案库存储接口 |

---

## 8. 交付标准

- [ ] 图谱解析器：支持顺序、并行、回环三种基本边
- [ ] Temporal 代码生成器：将解析结果编译为可执行的 Temporal Workflow
- [ ] 基本执行链路：意图 → 解析 → 生成 → Temporal 执行 → 状态回写
- [ ] AI 链路第一步：自然语言 → 结构化图谱（语义理解验证）
- [ ] 执行策略引擎：能力匹配 + 路径优化
- [ ] 变化监听器：微小/中等/重大三级响应
- [ ] 缓冲机制：意图暂存区 + 检查点
- [ ] 档案交接：节点间通过档案库传递数据
