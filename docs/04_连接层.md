# 板块四：连接层（事件总线 + 适配器 + 通信协议）

> 开发优先级：**P2 — 与板块三（工作流层）并行启动**
> 关联章节：v2.1 第三章（龙虾适配层）、第五章（连接层）、场景二（档案交接事件驱动）、场景三（龙虾纳管）

---

## 1. 板块职责

连接层是 ClawTeams 的"神经系统"，负责团队大脑与外部龙虾之间的所有通信，包括事件总线、适配器和开源通信协议。这是 ClawTeams 的系统边界——连接层以下是外部龙虾（OpenClaw等），不属于 ClawTeams。

---

## 2. 事件总线

### 技术选型

Kafka（持久化、高吞吐）或 WebSocket pub/sub（实时推送）

### 职责

- 龙虾与团队大脑之间的状态同步
- 龙虾与龙虾之间的直接协同
- 人工介入点的通知和响应
- 团队大脑内部各模块的状态变化传递
- 工作流节点间的档案交接事件传递

### 标准事件结构

```json
{
  "event_id": "唯一标识",
  "event_type": "task_completed | state_changed | human_required | cognitive_trigger | artifact_uploaded",
  "source_agent_id": "发起者",
  "team_id": "所属团队",
  "task_id": "关联任务",
  "payload": "结构化数据（非自然语言）",
  "timestamp": "时间戳",
  "requires_human": true/false,
  "artifact_ref": "关联档案引用"
}
```

### 事件类型清单

| 事件类型 | 触发时机 | 消费者 |
|---------|---------|--------|
| `task_completed` | 龙虾完成任务 | 工作流引擎、大脑状态模块 |
| `state_changed` | 意图/工作流状态变更 | 前端地图、相关龙虾 |
| `human_required` | 需要人工介入 | 通知系统、前端 |
| `cognitive_trigger` | 偏差超出容忍范围 | 认知模块 |
| `artifact_uploaded` | 成果上传到档案库 | 下游工作流节点 |
| `agent_registered` | 新龙虾创建注册 | 大脑账号模块 |
| `agent_state_sync` | 龙虾定时状态同步 | 大脑状态模块 |

## 3. 适配器

适配器存在于 ClawTeams 侧，**不修改龙虾本身**。

### ① 输出拦截器（Output Hook）

```
龙虾执行产出
  → Output Hook 捕获
  → 转换为标准化状态对象
  → 写入团队大脑
  → 龙虾本身无感知
```

### ② 上下文注入器（Context Injector）

```
任务下发前
  → Context Injector 从团队大脑拉取相关上下文
  → 注入到龙虾输入前缀
  → 龙虾带着团队上下文工作但无需感知
```

### ③ 事件订阅器（Event Subscriber）

```
龙虾注册时声明关心的事件类型
  → Event Subscriber 订阅相关事件
  → 收到新事件时触发龙虾重新评估执行计划
```

## 4. 龙虾⇔大脑通信协议（开源部分）

MCP 存在于龙虾内部（龙虾用 MCP 连接工具），但龙虾和团队大脑之间需要独立定义的协议，因为团队大脑是状态机图谱系统，不是 LLM。

### 注册协议

```json
{
  "agent_id": "...",
  "owner_id": "责任人",
  "capability_schema": "结构化能力声明（非自然语言）",
  "output_schema": "输出格式定义",
  "subscription_topics": ["订阅的事件类型"]
}
```

**能力声明必须是结构化枚举值，不接受自然语言描述。** 这是工作流生成器能力匹配准确的前提。

### 状态上报协议

```json
{
  "task_id": "...",
  "result": "结构化输出",
  "state": "枚举值: completed/failed/blocked/human_required",
  "requires_human": false,
  "cognitive_signal": "是否触发认知层判断",
  "artifact_ids": ["产出档案ID列表"]
}
```

### 事件订阅协议

```json
{
  "subscribe_topics": ["关心的事件类型"],
  "response_timeout": "秒",
  "timeout_escalation": "超时处理规则"
}
```

## 5. 龙虾三层分离与档案同步

每个龙虾的三层信息都通过连接层同步到大脑/档案库：

| 层 | 同步内容 | 同步频率 | 存储位置 |
|---|---------|---------|---------|
| 技能层 | system prompt、工具链、参数、能力声明 | 配置变更时 | Agent Registry |
| 环境层 | Docker镜像版本、依赖清单 | 环境变更时 | Environment Store |
| 数据上下文层 | 执行历史、业务数据引用 | 实时 | 团队大脑（域内隔离） |

### 风险场景应对

| 风险 | 连接层职责 |
|------|----------|
| 员工电脑丢失 | 三层信息全部在云端有备份，新机器一键恢复 |
| 员工离职 | 连接层配合大脑执行 owner 变更，龙虾不中断 |
| 网络断开 | 允许本地临时缓存，重新联网后强制同步 |
| 龙虾异常 | Output Hook 检测异常，触发 state_changed 事件 |

---

## 6. 与其他板块的接口

| 对接板块 | 接口内容 |
|---------|---------|
| 团队大脑 | 龙虾注册/注销、状态上报写入、事件分发 |
| 工作流层 | 任务下发、执行状态回收、档案交接事件 |
| 前端 | WebSocket 事件流推送（地图实时更新） |
| 外部龙虾 | 通信协议（注册、上报、订阅）— ClawTeams 系统边界 |

## 7. 交付标准

- [ ] 事件总线基础设施就绪（Kafka 或 WebSocket）
- [ ] 标准事件结构定义并发布
- [ ] Output Hook：捕获龙虾输出 → 标准化 → 写入大脑
- [ ] Context Injector：从大脑拉取上下文 → 注入龙虾输入
- [ ] Event Subscriber：事件订阅 → 触发龙虾响应
- [ ] 通信协议 v1：注册协议 + 状态上报协议 + 事件订阅协议
- [ ] 协议验证工具：测试龙虾实现是否符合协议标准
- [ ] 龙虾三层信息同步机制
- [ ] 断网缓存 + 重连同步
